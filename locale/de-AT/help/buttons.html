Button Liste

Back

Type: Button
Standard Status: Versteckt
Anpassbar: NEIN

Dies ist ein sehr einfacher "Zurück" Button, der eine Seite in der Browser-History zurückspringt, genau so wie der "Zurück" Button in der Navigations-Symbolleiste. Anders als der "Zurück" Button, in der Navigations-Symbolleiste, hat dieser Button keine Dropdown-Liste, mit den zuletzt gewählten Seiten, hat kein Icon und wird nicht deaktiviert, wenn es keine Seite zum zurückgehen gibt. Wie gesagt, es ist nur ein einfacher "Zurück" Button.

Forward

Type: Button
Standard Status: Versteckt
Anpassbar: NEIN

Dies ist ein sehr einfacher "Weiter" Button, ähnlich wie der einfache "Zurück" Button, der oben beschrieben wurde. Alle Einschränkungen, die dort genannt wurden, gelten auch hier.

Reload

Type: Button
Standard Status: Versteckt
Anpassbar: NEIN

Dies ist ein sehr einfacher "Neu laden" Button, der zum einfachen "Zurück" Button und zum einfachen "Weiter" Button gehört. Anders als bei den oben beschriebenen Buttons hat dieser Button bereits alle Funktionalitäten wie der zugehörige Button auf der Navigations-Symbolleiste, was bedeutet das Sie ihn bei gedrückter Shift-Taste anklicken können, um das neu Laden der Seite, unabhängig vom Browser-Cache, zu erzwingen oder einfach klicken für ein normales neu Laden.

Stop

Type: Button
Standard Status: Versteckt
Anpassbar: NEIN

Ein sehr einfacher "Stopp" Button, um die Gruppe zu vervollständigen. Dieser Button beobachtet nicht die Aktivität im Browser und ist deshalb auch aktiv, wenn der Navigator im Moment nichts tut (zu dieser Zeit hat das Anklicken aber absolut keinen Effekt).

Separator

Type: Separator
Standard Status: Versteckt
Anpassbar: NEIN

Dieser Abstandshalter kommt zwischen die Navigations-Buttons und den Home-Button (oder wo auch immer sie ihn an der rechten Seite der Navigations-Buttons anordnen). Es gibt mehrere Abstandshalter in der PrefBar Button-Liste, einen für jede Gruppe von Elementen. Sie brauchen ein wenig Platz auf der Toolbar, aber sie sorgen dafür, dass die Button-Anordnungen besser aussehen. Deshalb ist es empfehlenswert, die Abstandshalter aktiviert zu lassen, es sei denn Sie haben einen sehr kleinen Monitor (oder wollen eine große Anzahl von Buttons auf einmal aktiviert haben).

Home

Type: Button
Standard Status: Versteckt
Anpassbar: NEIN

Obwohl dieser Button nicht so schön aussieht, wie der richtige "Startseite" Button (er hat das Icon nicht, zumindest noch nicht) bringt ein Klick sie genau so schnell zu ihrer Startseite.

Separator

Type: Separator
Standard Status: Versteckt
Anpassbar: NEIN

Ein Abstandshalter, der auf die rechte Seite des Home-Buttons gehört. Sie können ihn (natürlich) an jede beliebige Stelle auf der Toolbar verschieben.

Fonts

Type: Checkbox
Standard Status: Versteckt
Anpassbar: NEIN

Dies schaltet die "Dokumenten erlauben, andere Schriftarten zu verwenden" Eigenschaft ein oder aus. Wenn Sie eine Webseite mit wirklich schrecklichen Schrift-Einstellungen finden, dann stellen Sie dies ab, um die Seite stattdessen mit normalen Schriften angezeigt zu bekommen.

Colors

Type: Checkbox
Standard Status: Sichtbar
Anpassbar: NEIN

Vielleicht die praktischste Checkbox von allen, diese stellt die von Autor festgelegten Farben ab und zeigt alles mit den Standardfarben an. Nur weil manche schwachsinnigen meinen, dass gelber Text auf einem roten Hintergrund schön aussieht, muss das nicht heißen, dass Sie das so lesen sollen. Von jetzt an müssen Sie nicht mehr.

System Colors

Type: Checkbox
Standard Status: Versteckt
Anpassbar: NEIN

Wechselt zwischen den Farben, die im Einstellungs-Dialog festgelegt wurden, und den Systemfarben. Nur sinnvoll, wenn die "Colors" Checkbox deaktiviert wird.

Images

Type: Checkbox
Standard Status: Sichtbar
Anpassbar: NEIN

Dies schaltet die Grafik-Anzeigeregeln von "Alle Grafiken akzeptieren" (wenn aktiviert) nach "Keine Grafiken laden" (wenn deaktiviert). Sie müssen dies entweder vor dem Laden der Seite einstellen, oder Sie müssen die Seite, nach dem Ändern der Einstellung, neu Laden. (Ich weiß nicht, warum diese Einstellung nicht automatisch funktioniert, wie die "Fonts" und "Farben" Einstellungen. Es ist vielleicht ein Bug, aber ich kenne die Nummer nicht.)

Animation

Type: Checkbox
Standard Status: Versteckt
Anpassbar: NEIN

Dies deaktiviert/aktiviert animierte Bilder. Die Einstellung hat keinen Einfluss auf andere Animationen, wie JavaScript oder Flash. Sie müssen diese Einstellung entweder vor dem Laden einer Seite festlegen, oder die Seite neu laden, wenn die Einstellung verändert wurde.

JavaScript

Type: Checkbox
Standard Status: Sichtbar
Anpassbar: NEIN

Aktiviert oder deaktiviert JavaScript. Dies kann manchmal sehr nützlich sein. Beachten Sie, dass diese Einstellung --wie bei allen alle Eigenschaften-- global vorgenommen wird. Deshalb deaktivert das Abstellen von JavaScript in einem Browser-Fenster oder -Tab die JavaScript Unterstützung von Mozilla vollständig.

Java

Type: Checkbox
Standard Status: Versteckt
Anpassbar: NEIN

Aktiviert oder deaktiviert Java. Persönlich bevorzuge ich es Java die meiste Zeit deaktiviert zu lassen und nur dann zu aktivieren, wenn ich ein Java Applet bemerke das ich wirklich brauche. So bleibe ich von Java basierter Werbung oder der Wartezeit, beim Laden von Java auf einer Seite, die Java benötigt, erspart.

Proxies

Type: Checkbox
Default Status: Versteckt
Anpassbar: NEIN

Diese Checkbox schaltet die Proxy-Einstellungen zwischen "Direkte Verbindung zum Internet" (wenn deaktiviert) und der letzten Proxy-Einstellung, die eingestellt war, bevor Sie den Haken in der Checkbox entfernt haben (wenn aktiviert).

Anpassbar: NEIN

Options: No Proxies (Proxies sind deaktiviert) Manual Proxy (Manuelle Proxy Konfiguration wird verwendet) Automatic Proxy (Automatische Proxy Konfigurations URL wird genutzt)

User Agent

Type: Menulist
Standard Status: Sichtbar
Anpassbar: JA

Dies erlaubt es, den User Agent String von Mozilla schnell zu ändern, was praktisch ist, wenn man Seiten betrachten will, die alle Browser, außer IE, blockieren. Das funktioniert für alle serverseitigen und einige clientseitige Prüfungen.

Die PrefBar setzt den User Agent automatisch auf den (richtigen) Standardwert, wenn Mozilla beendet wird. Dies verhindert Probleme mit Bug 83376 (der dafür sorgt das Mozilla crasht) und stellt sicher, dass nicht vergessen wird, dass der UA-String geändert wurde, und dieser somit nicht für 3 Wochen verändert bleibt.

Optionen: Real UA [Der korrekte Wert für das Mozilla-Build das Sie nutzen] Moz 1.0 Win98 [Mozilla/5.0 (Windows; U; Win98; en-US; rv:1.0.0) Gecko/20020530] NS 4.7 Mac [Mozilla/4.78 (Macintosh; U; PPC)] NS 6.2 Lin [Mozilla/5.0 (X11; U; Linux i686; en-US; rv:0.9.4) Gecko/20011022 Netscape6/6.2] IE 5.0 Mac [Mozilla/4.0 (compatible; MSIE 5.0; Mac_PowerPC)] IE 6.0 WinXP [Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1)]
